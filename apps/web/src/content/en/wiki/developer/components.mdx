This page showcases all the custom components available for use in your documentation. These components are inspired by modern documentation frameworks like MkDocs Material and Starlight.

## Cards

### Basic Card

Use `Card` to highlight important information or features.

<Card title="Getting Started" icon={<DocIcon name="rocket" />}>
  This is a basic card component that can be used to display important information with an optional icon and title.
</Card>

### Card Grid

Use `CardGrid` to display multiple cards in a responsive grid layout.

<CardGrid cols={{ base: 1, sm: 2, lg: 3 }}>
  <Card title="Fast" icon={<DocIcon name="bolt" />}>
    Lightning-fast performance optimized for modern browsers.
  </Card>
  <Card title="Secure" icon={<DocIcon name="lock" />}>
    Built with security best practices in mind.
  </Card>
  <Card title="Scalable" icon={<DocIcon name="trending-up" />}>
    Designed to grow with your needs.
  </Card>
</CardGrid>

## Code Tabs

Use `CodeTabs` for multi-language code examples, with `CodeTab` for each language.

Language icons are automatically added based on the language specified.

<CodeTabs>
  <CodeTab label="JavaScript" language="javascript">
    ```js
    function greet(name) {
      return `Hello, ${name}!`;
    }
    ```
  </CodeTab>
  <CodeTab label="TypeScript" language="typescript">
    ```ts
    function greet(name: string): string {
      return `Hello, ${name}!`;
    }
    ```
  </CodeTab>
  <CodeTab label="Python" language="python">
    ```bash
    def greet(name):
        return f"Hello, {name}!"
    ```
  </CodeTab>
</CodeTabs>

## File Tree

Use `FileTree` to visualize project structure with support for highlighting, comments, placeholders, and collapsible folders.

### Basic Usage

<FileTree>
- src/
  - components/
    - Button.tsx
    - Card.tsx
  - utils/
    - helpers.ts
  - index.ts
- package.json
- README.md
</FileTree>

### Highlighted Files

Use `**bold**` syntax to highlight important files or folders:

<FileTree>
- src/
  - **index.ts** // main entry point
  - components/
    - Button.tsx
  - utils/
- **package.json**
- README.md
</FileTree>

### Comments

Add comments after filenames using `//` to provide additional context:

<FileTree>
- src/
  - index.ts // the main entry point
  - components/
    - Button.tsx // reusable button component
    - Card.tsx // with fancy styling
  - config.ts // uses dotenv for environment variables
- package.json // project dependencies
</FileTree>

### Placeholders

Use `...` or `…` to indicate additional files:

<FileTree>
- src/
  - components/
    - Button.tsx
    - Card.tsx
    - ...
  - utils/
- node_modules/
  - …
- package.json
</FileTree>

### Full Example

Combining all features:

<FileTree>

  - components/

## Admonitions

Enhanced blockquotes with multiple types and custom titles.

These components can optionally be written with purely markdown syntax:

> [!NOTE] "Custom title" 
> This is a note admonition. Use it for general information.

> [!TIP] "Helpful Tip"
> This is a tip admonition. Use it for helpful suggestions.

> [!IMPORTANT] "Important Information"
> This is important information that users should know.

> [!WARNING] 
> This is a warning. Use it to alert users about potential issues.

> [!CAUTION]
> This is a caution. Use it for critical warnings.

Or if you want to use JSX:

<BlockQuote type="note" title="Custom Note">
  This is a custom note admonition with a personalized title.
</BlockQuote>

Use `Details` for collapsible content sections.

  ```js
  const config = {
    advanced: true,
    options: ['a', 'b', 'c']
  };
  ```
</Details>

<Details summary="Another collapsible section" defaultOpen>
  This section is open by default because of the `defaultOpen` prop.
</Details>
